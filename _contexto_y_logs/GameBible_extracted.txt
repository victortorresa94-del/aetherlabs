00. VISION GENERAL Y PROPOSITO
La web de Aether Labs ES un juego de Game Boy. El usuario no navega una web corporativa — juega un videojuego retro pixel art que le guia desde su problema hasta la solucion, y al final reserva una reunion con Victor Torres (Aether Labs).
El juego cumple tres objetivos comerciales simultaneos:
Diferenciacion radical — nadie en el mercado espanol de servicios IA tiene una web asi
Prueba de capacidad — si esta web funciona asi, el cliente piensa: si me hacen esto a mi, voy a quedar de puta madre
Claridad sin ruido — el juego fuerza al usuario a identificarse con su problema antes de ver servicios
Los tres Labs del juego
CREATIVE LAB — Cyan #00E5FF — Para empresas que necesitan ser vistas. Mundo: El Vacio.
SYSTEMS LAB — Azul #0088FF — Para empresas que quieren automatizar e implementar IA. Mundo: El Bucle.
SCHOOL LAB — Verde #00FF41 — Para personas y equipos que quieren aprender IA. Mundo: El Umbral.
01. NARRATIVA Y UNIVERSO
El universo Aether
Aether es el eter — el espacio entre dimensiones, lo invisible que existe antes de que las cosas tomen forma. Aether Labs convierte eso etero en algo real y palpable. La web lo demuestra viviendo el proceso.
El jugador entra en el eter. Su mision es atravesarlo y llegar al otro lado — donde las cosas se vuelven reales y ejecutables.
El arco narrativo completo
BOOT SCREEN — El logo de Aether Labs aparece. El mundo se enciende.
LAB SELECT — Tres puertas. El jugador elige su mundo.
ENTRADA AL MAPA — Aparece en el borde del mundo. Oscuro, caotico o silencioso.
EXPLORACION — Encuentra NPCs. Escucha sus problemas. Reconoce el suyo.
AVANCE — El mundo cambia a su paso. Se ilumina, se ordena, se abre.
EL COFRE — Llega al final. Abre el cofre. Ve la solucion concreta.
CTA — Mision completada. Reserva reunion. Sale con algo real.
Los tres mundos
El Vacio — Creative Lab
Ciudad nocturna abandonada donde las empresas existen pero son invisibles. Pantallas apagadas, logos borrosos, senales sin luz. A medida que el jugador avanza y activa los NPCs, la ciudad se va iluminando.
El Bucle — Systems Lab
Fabrica industrial atrapada en un loop eterno. Los mismos informes, las mismas tareas manuales, los mismos procesos repetidos. Engranajes que giran pero no avanzan. El jugador rompe el loop conectando sistemas.
El Umbral — School Lab
Escuela abandonada llena de potencial dormido. Puertas cerradas, pizarras con informacion valiosa, ordenadores con demos. El jugador abre puertas y activa el conocimiento dormido.
Los NPCs — reglas
Los NPCs no son personajes de fantasia. Son arquetipos reales de clientes de Aether Labs. Cada uno verbaliza su problema en primera persona. Maximo 3 lineas de dialogo. Lenguaje coloquial, sin tecnicismos. Nunca venden, nunca explican servicios.

03. MECANICAS DE JUEGO
Core Loop
1. Jugador se mueve por el mapa
2. Encuentra NPCs y los activa (Espacio cuando esta cerca)
3. Lee el dialogo (se identifica con el problema)
4. Barra de progreso HUD avanza
5. Camino al cofre se desbloquea cuando progreso = 100%
6. Llega al cofre, lo activa
7. Ve los servicios del Lab
8. Hace clic en HABLAR CON VICTOR
Movimiento del jugador
Motor: Phaser Arcade Physics (AABB collision, sin gravedad). Teclas: WASD o flechas. Velocidad: 120px/s. Direcciones: 4 (arriba, abajo, izquierda, derecha). Sin diagonal en v1.
Colisiones con capa collision del tilemap. El sprite siempre mira la ultima direccion de movimiento cuando esta quieto.
Interaccion con NPCs
Zona de proximidad: 48px alrededor del NPC (overlap zone)
Indicador: burbuja '!' sobre el NPC cuando jugador entra en zona (animacion bounce)
Activacion: Espacio o click sobre el NPC
Solo un NPC activo a la vez
NPC activado muestra checkmark verde y deja de mostrar el '!'
Sistema de progreso
La barra del HUD representa NPCs activados / total NPCs del mapa.
El cofre SOLO se puede abrir cuando la barra llega al 100%. Si el jugador llega antes: cofre con candado + mensaje ENCUENTRA TODAS LAS SENALES.
Al completar: texto TODAS LAS SENALES ENCONTRADAS! aparece 1.5s y desaparece + fanfare.
Estructura del tilemap
Cada mapa: 25x18 tiles de 32x32px = 800x576px. Offset vertical de 24px para el HUD.
Capas (orden de renderizado): background / decoracion / collision (invisible, solo logica) / objects (NPCs, cofre) / foreground.
Punto de inicio jugador: siempre borde inferior del mapa. Cofre: siempre extremo superior.
Objetos interactivos adicionales
Pizarras, pantallas, ordenadores y documentos en el mapa son activables con Espacio. Muestran informacion sobre herramientas, servicios o casos de trabajo de Aether Labs. NO cuentan para la barra de progreso — son bonus de contenido.
Mobile
En pantallas menores de 768px: mensaje ESTE JUEGO ES MEJOR EN ESCRITORIO + opcion CONTINUAR EN MOVIL con D-pad tactil (4 botones 44px en esquina inferior izquierda, boton de accion en derecha).
04. DISENO VISUAL — ESTETICA Y ARTE
Estetica global
Game Boy Color / SNES era. Pixel art de 16 bits con paleta restringida, sprites pequenos y tilemaps simples. Debe parecer autentico, no una parodia.
Referencias: Game Boy Color modo oscuro / Zelda Links Awakening (vista top-down) / Hyper Light Drifter (paleta de neones sobre negro) / EarthBound (dialogos con personalidad).
Resolucion y escala
Canvas interno: 800x600px (coordenadas del juego)
Tiles: 32x32px
Sprites personaje: 16x24px nativos, escalados a 32x48px en juego
Sprites NPC: 16x16px nativos, escalados a 32x32px
Canvas escala con CSS para ocupar 100% del viewport (letterbox negro)
Paleta de colores global

Texto UI:     #FFFFFF (principal) | #AAAAAA (secundario)
UI bordes:    color del Lab activo
Progreso vacio: #333344

Jerarquia de tamaños: titulos 12-14px / dialogo NPC 8px / labels HUD 7-8px / cofre principal 10-12px / cofre secundario 7-8px / botones 8-9px.

Cursor: cursor: none + sprite pixel art de mano 16x16px generado como data URL
05. DISENO DEL PERSONAJE PRINCIPAL
Concepto
Avatar neutral sin nombre ni historia. Es el propio usuario convertido en sprite. Humanoid minimalista en pixel art Game Boy. Sin rasgos faciales definidos (vista top-down).
Dimensiones del spritesheet
Sprite base: 16x24px. Escala en juego: 2x (32x48px renderizados)
Frames: 4 frames por direccion x 4 direcciones = 16 frames total
Formato: spritesheet 64x96px (4 columnas x 4 filas)
Fila 0: caminar hacia abajo (sur) — frames 0,1,2,3
Fila 1: caminar hacia arriba (norte) — frames 4,5,6,7
Fila 2: caminar hacia la izquierda (oeste) — frames 8,9,10,11
Fila 3: caminar hacia la derecha (este) — frames 12,13,14,15
Descripcion visual del sprite
Cabeza: circulo 6x6px, color piel neutro #e8c8a0
Cuerpo: rectangulo 4x8px — color cambia por Lab: Creative=#005566 / Systems=#002244 / School=#002200
Detalles del color Lab: Creative=#00E5FF / Systems=#0088FF / School=#00FF41
Brazos: 1px a cada lado del cuerpo, 4px de largo
Piernas: 2 columnas de 2px, 6px de largo
Sombra: elipse plana debajo, color negro, opacidad 0.3
Animaciones del personaje
Caminar: 4 frames por direccion, 150ms por frame = 600ms ciclo completo. Frame 0: posicion neutral. Frame 1: paso adelante izquierdo. Frame 2: posicion neutral. Frame 3: paso adelante derecho. Bob vertical de 2px en frames 1 y 3.
Idle: respiracion — escala Y del cuerpo 1.0 a 0.97 y vuelta, ciclo 2 segundos.
Activar NPC: giro hacia el NPC + animacion brazo sube 3px, 2 frames 200ms cada uno.
Cofre abierto: salto translateY -8px en 200ms + vuelta en 200ms.
Camara
camera.setBounds(0, 0, mapWidth, mapHeight)
camera.startFollow(player, true, 0.1, 0.1) — lerp suave 0.1
camera.setZoom(1) — sin zoom en desktop
El HUD no se mueve con la camara — es una UI layer con setScrollFactor(0).

document.addEventListener('click', () => {
  if (!audioCtx) audioCtx = new AudioContext();
}, { once: true });
Estructura de ganancia: MasterGainNode (0.0-1.0) / MusicGainNode (0.7) / SFXGainNode (1.0).
Funcion base para reproducir notas
function playNote(freq, duration, type='square', gain=0.3, delay=0) {
  const osc = audioCtx.createOscillator();
  const gainNode = audioCtx.createGain();
  osc.type = type;
  osc.frequency.setValueAtTime(freq, audioCtx.currentTime + delay);
  gainNode.gain.setValueAtTime(gain, audioCtx.currentTime + delay);
  gainNode.gain.exponentialRampToValueAtTime(0.001, audioCtx.currentTime + delay + duration);
  osc.connect(gainNode); gainNode.connect(audioCtx.destination);
  osc.start(audioCtx.currentTime + delay);
  osc.stop(audioCtx.currentTime + delay + duration);
}

Configuracion Phaser
const config = {
  type: Phaser.AUTO,
  width: 800, height: 600,
  backgroundColor: '#050508',
  pixelArt: true,       // CRITICO: sin antialiasing
  roundPixels: true,    // Evita blur en movimiento
  physics: {
    default: 'arcade',
    arcade: { gravity: { y: 0 }, debug: false }
  },
  scale: { mode: Phaser.Scale.FIT, autoCenter: Phaser.Scale.CENTER_BOTH },
  scene: [PreloadScene, BootScene, LabSelectScene, CreativeMapScene,
          SystemsMapScene, SchoolMapScene, ChestScene, CTAScene]
};
